Introducere în CSS

CSS (Cascading Style Sheets, foi de stil în cascadă) – este un limbaj pentru descrierea și modificarea aspectului elementelor. HTML este folosit pentru a defini structura și semantica conținutului unui document, în timp ce CSS este folosit pentru stil și poziționare.

Sintaxă

Un bloc de cod CSS e definit ca fiind o regulă și e compus dintr-un selector și un grup de anunțuri între acolade.

Selectorul specifică browserului la ce elemente să aplice stilurile din această regulă. Fiecare declarație e formată dintr-o pereche proprietate: valoare;, unde proprietatea și valoarea ei sunt separate prin două puncte și un spațiu și trebuie să se termine cu punct și virgulă. Ele descriu exact ceea ce trebuie făcut cu stilurile elementului.

Conectarea stilurilor

Există trei moduri de a adăuga stiluri la un document HTML: stiluri de tip inline (inline styles), stiluri încorporate (embedded stylesheet) și stiluri externe (external stylesheet). Fiecare metodă are avantaje, dezavantaje, limitări și un domeniu de aplicare.

    Stiluri de tip inline

    Stilurile de tip inline sunt scrise direct în tag, în atributul style. Acestea nu pot fi scalate, sunt greu de redefinit și reutilizat, de aceea sunt folosite în cazuri foarte rare. De exemplu, pentru stilurile dinamice care sunt instalate în timpul execuției codului JavaScript, când încă nu se știe care va fi valoarea proprietății.

    Exemplu :

    (p style="color:blue; font-size:24px;")Acest text va fi albastru.(/p)

    Stiluri încorporate

    Folosind aceeași abordare, regulile CSS sunt adăugate în interiorul tag-ului (style) de la începutul documentului. Ele nu pot fi reutilizate pe alte pagini și sunt greu de scalat și întreținut.

    Exemplu :

    (head)
    (style type="text/css")
    p { color: blue; font-size: 24px; }
    (/style)
    (/head)

    Interesant !!!

    Uneori, această abordare este folosită pentru a optimiza viteza de redare a paginii prin adăugarea celor mai importante stiluri direct în documentul HTML. Această tehnică avansată se numește Critical CSS.

    Stiluri externe

    Codul CSS extern este ușor de scalat, întreținut și reutilizat pe alte pagini, motiv pentru care reprezintă un standard pentru încorporarea stilurilor. În proiect se creează un fișier de stil separat cu extensia .css și se include în documentul HTML.

    • Un dosar css se creează la același nivel cu index.html, iar în interiorul acestuia se află un fișier de stil styles.css.
    • În tag-ul (head), folosind tag-ul (link), se conectează un fișier creat anterior.
    • Atributul href specifică calea către fișierul de stiluri relativ la documentul HTML.
    • Atributul rel specifică tipul documentului conectat – stylesheet (foaia de stil).

    Util !

    Puteți include oricâte fișiere CSS doriți, adăugând pentru fiecare tag-ul (link).

Selectori

Cu ajutorul a diferiți selectori, puteți selecta un element sau un grup de elemente. Selectori există destul de mulți, de aceea, haideți să-i analizăm pe cei de bază care sunt folosiți, de altfel, cel mai des.

  • Selectorul de element (tag)

    Descrie cărui tip de elemente îi va fi aplicată regula CSS. Selectorul de elemente nu este suficient de specific pentru a stiliza elemente individuale, așa că este cel mai des folosit pentru a stiliza toate tag-urile de același tip pe pagină. De exemplu, dacă trebuie să înlăturați sublinierea vizuală din toate link-urile documentului.

    /* Se aplică tuturor paragrafelor de pe pagină */
    p {
    font-size: 24px;
    }
    /* Se aplică tuturor link-urilor de pe pagină */
    a {
    text-decoration: none;
    }

  • Selectorul de clasă

    În dezvoltarea modernă acesta este un selector de bază. Este utilizat împreună cu atributul global class. Numele clasei este specificat numai în engleză și trebuie să fie un substantiv care descrie ce fel de element este. Pentru a vă extinde vocabularul, utilizați Referințele, Glosarul de termeni și Glosarul de cuvinte și abrevieri.

    CSS folosește selectorul de clasă pentru a stiliza unul sau mai multe elemente cu aceeași valoare a atributului class (numele clasei). Într-un selector, înainte de numele clasei, se pune un punct .

    /* Se aplică tuturor tag-urilor cu clasa title */
    .title {
    font-weight: 500;
    }

    /* Se aplică tuturor tag-urilor cu clasa text */
    .text {
    color: brown;
    font-size: 18px;
    }

    /* Se aplică tuturor tag-urilor cu clasa link */
    .link {
    text-decoration: none;
    }

    În numele clasei sunt folosite doar litere mici și liniuțe. De exemplu, list, list-item, logo-image etc. Sublinierea list_item, majusculele ListItem și cifrele listitem27 sunt considerate de prost gust.

    Compoziția clasei

    Un element poate avea mai multe clase, iar în acest caz acestea sunt scrise folosind un spațiu.

    (element class="value1 value2 value3")(/element)

    Acesta poate fi folosit pentru a combina și reutiliza stiluri. Haideți să creăm marcajul și stilurile pentru o componentă de notificări.

    In HTML

    (p class="alert success) Reîncărcarea contului a fost finalizată (/p)

    (p class="alert warning")Atenție, tarifele se vor schimba(/p)

    (p class="alert error")Eroare tranzacție(/p)

    In CSS

    În stiluri descriem câteva reguli comune tuturor alertelor și specifice fiecărui tip.

    p/* Stiluri comune pentru toate tipurile de alerte */
    .alert {
    font-size: 24px;
    font-weight: 500;
    }

    /* Stiluri specifice pentru fiecare tip */
    .success {
    color: green;
    }

    .warning {
    color: orange;
    }

    .error {
    color: red;
    }

  • Selectorul de ID

    Ca și în cazul unei clase, tag-urilor li se poate desemna un atribut global id. Diferența este că valoarea id trebuie să fie unică pe pagină, adică nu pot exista două elemente cu aceleași valori id.

    (h1 id="title")Titlul paginii(/h1)

    În selectorul CSS, înainte de numele identificatorului, este plasat simbolul hash #.

    #title {
    font-weight: 500;
    color: orange;
    }

    Util !!!

    Nu utilizați selectori de ID pentru foile de stil. Regula valorii unice face imposibilă reutilizarea stilurilor. Identificatorii sunt destinați pentru "ancore".

  • Selectorul descendent

    Selectorul descendent, sau selectorul contextual, atribuie stiluri descendenților unui element, indiferent de nivelul imbricare.

    (a href="")Citiți blogul nostru!(/a)
    (ul class="social-links")
    (li)
    (a href="")Twitter
    (/li)
    (li)
    (a href="")Instagram
    (/li)
    (li)
    (a href="")Facebook
    (/li)
    (/ul)

    De exemplu, în loc să setați un stil pentru toate link-urile, trebuie doar să schimbați stilurile care se află în lista cu clasa social-links.

    /* ❌ Se aplică tuturor link-urilor din document */
    a {
    color: blue;
    text-decoration: none;
    }

    /* ✅ Se aplică numai link-urilor care se află în interiorul tag-ului cu clasa social-links */
    .social-links a {
    color: blue;
    text-decoration: none; }

  • Selectorul de copil

    Acesta este un alt selector comun care permite selectarea nemijlocită a copilului în interiorul elementului părinte. Să folosim marcajul pentru un meniu de magazin cu mai multe niveluri, în care trebuie să stilizăm tag-urile

  • din listă cu clasa menu, dar nu și cu clasa submenu.

    • Exemplu

    • Calculatoare și accesorii
      • Procesoare
      • Monitoare
      • Plăci video
    • Aparate electrocasnice
      • Frigidere
      • Televizoare
      • Mașini de spălat
    • Produse de uz casnic
      • Fotolii
      • Saltele
      • Șemineuri electrice

    Dacă folosiți un selector descendent (contextual), atunci regula CSS va fi aplicată tuturor tag-urilor (li), iar fiecare va avea un chenar roșu.

    /* ❌ Nu rezolvă problema */
    .menu li {
    border: 1px solid tomato;
    }

    Selectorul de copil rezolvă această problemă, selectând doar acele (li) care sunt copii (prima imbricare) din lista ul.menu.

    /* ✅ Exact ceea ce aveți nevoie */
    .menu > li {
    border: 1px solid tomato;
    }

  • Selectorul de atribut

    Vă permite să aplicați stiluri la elementele cu un anumit atribut sau valoare.

    Exemple

    • Exemplu 1

      element[atribut]

      Aplică stiluri la toate elementele care au un atribut specificat.

      htmlreference.io cssreference.io YouTube

      <a href="https://htmlreference.io" title="Referință pentru tag-uri" > htmlreference.io </a>

      <a href="https://cssreference.io" title="Referință pentru stiluri" > cssreference.io </a>

      <a href="https://www.youtube.com">YouTube </a>

      Selectează toate link-urile cu atributul title.

      a[title] { color: teal; }

    • Exemplu 2

      element[atribut="valoare"]

      Aplică stiluri la toate elementele care au un atribut cu valoarea specificată.

      Selectează toate link-urile cu atributul href și cu valoarea corespunzătoare.

      /* Formatarea neobișnuită a acoladelor este ceva normal */
      a[href="https://htmlreference.io"] { color: teal; }
      a[href="https://cssreference.io"] { color: orange; }
      a[href="https://www.youtube.com"] { color: tomato; }

Pseudo-clase de stare

Selectorii de stare sunt folosiți pentru a aplica stiluri elementelor interactive atunci când are loc un anumit eveniment, cum ar fi trecerea cursorului peste un link sau când elementul devine focalizat de la tastatură.

Pseudo-clasa definește o stare specifică a elementului și este atașată unui selector de tag, clasă etc. Nu există spațiu între selector și pseudo-clasă pentru a arăta că acestea sunt legate între ele. Dacă adăugați un spațiu, atunci pseudo-clasa se va aplica tuturor elementelor din document.

selector:pseudo-clasă {
/* Proprietăți */
}

  • Pseudo-clasa :hover

    Aceasta este activată atunci când cursorul mouse-ului se află în cadrul unui element, cum ar fi când treceți mouse-ul peste un link sau orice alt element.

    Exemplu

    • Exemplu 1

      Puteți găsi referințe grozave, accesând link-ul.

      Puteți găsi referințe grozave, <a class="link" href="https://htmlreference.io" >accesând link-ul </a >.

      Să adăugăm un stil de bază și un stil de focalizare unui link.

      .link {
      color: teal;
      }
      .link:hover {
      color: tomato;
      }

    • Exemplu 2

      Pseudo-clasele pot fi aplicate oricăror selectori, inclusiv celor de context.

      • Twitter
      • Instagram
      • Facebook

      <ul class="social-links">
      <li> <a class="link" href="https://twitter.com">Twitter</a> </li>
      <li> <a class="link" href="https://www.instagram.com" >Instagram</a > </li>
      <li> <a class="link" href="https://www.facebook.com" >Facebook</a > </li>
      </ul>

      Să adăugăm un stil de bază și un stil de focalizare unui link.

      .social-links .link {
      color: teal;
      }

      .social-links .link:hover {
      color: tomato;
      }

    Util !!!

    Un astfel de selector arată "Când treceți cu mouse-ul peste elementele cu clasa link care se află în interiorul unui element cu clasa social-links". Adică, selectorii sunt citiți de la dreapta la stânga.

  • Pseudo-clasa :focus

    Este activată atunci când un element interactiv (link, buton, câmp de formular) este focalizat în momentul în care navighează pe pagină folosind tastatura (tasta Tab).

    Exemplu

    • Twitter
    • Instagram
    • Facebook
    HTML

    (a href="")Citiți blogul nostru!(/a)
    (ul class="social-links")
    (li)
    (a href="")Twitter
    (/li)
    (li)
    (a href="")Instagram
    (/li)
    (li)
    (a href="")Facebook
    (/li)
    (/ul)

    Să adăugăm un stil de bază și un stil de focalizare link-ului din interiorul unei liste.

    CSS

    .ssocial-media-links .link {
    color: tomato;
    }

    .social-media-links .link:focus {
    color: green;
    }

    Pentru a ține cont de hover și de focalizare, trebuie să stilizați ambele stări simultan. Acest lucru se face printr-o simplă enumerare a selectorilor.

    CSS

    .social-media-links .link {
    color: tomato;
    }

    /* Doi sau mai mulți selectori sunt separați prin virgulă */
    .social-media-links .link:hover,
    .social-media-links .link:focus {
    color: green;
    }

  • Pseudo-clasa :active

    Aceasta apare atunci când elementul este activat. De exemplu, un link devine activ dacă treceți cursorul peste el și dați click. Deși aproape orice element poate deveni activ, pseudo-clasa :active este folosită în primul rând pentru link-uri și butoane.

    CSS

    .social-links .link {
    color: teal;
    }

    .social-links .link:hover,
    .social-links .link:focus {
    color: orange;
    }

    .social-links .link:active {
    color: red;
    }

  • Pseudo-clasa :visited

    Această pseudo-clasă este activată pe link-urile care au fost deja vizitate. În mod implicit, link-urile sunt afișate în albastru și devin violete atunci când sunt vizitate.

    CSS

    .social-links .link {
    color: teal;
    }

    .social-links .link:hover,
    .social-links .link:focus {
    color: orange;
    }

    .social-links .link:active {
    color: red;
    }

    .social-links .link:visited {
    color: green;
    }

  • Hover și tabele

    Pseudo-clasa :hover nu trebuie aplicată link-urilor, dar poate fi adăugată și altor elemente ale documentului. De exemplu, la rândurile sau celulele unui tabel, astfel încât acestea să schimbe culoarea de fundal în momentul în care rotiți cursorul mouse-ului.

    Exemplu

    • HTML
      • <table class="schedule">>
      • <tbody class="schedule-body">
    • Număr Traseu Timp de călătorie
      433 Cluj-Napoca - Sibiu 5 ore
      701 Vaslui - Oradea 7 ore
      258 Arad - Sibiu 4 ore
    • CSS
      .schedule-body > tr:hover {
      background-color: tomato;
      color: white;
      }

      /* Marcaj tabel */
      .schedule {
      border-collapse: collapse;
      }

      .schedule th,
      .schedule td {
      padding: 8px;
      border: 1px solid #212121;
      }

  • Proprietatea cursor

    Aceasta setează tipul cursorului mouse-ului atunci când acesta se află în cadrul elementului. Aspectul cursorului depinde de sistemul de operare și de setările acestuia. Schimbarea tipului de cursor este o idee rea, încât va deruta utilizatorul doar când va vedea altceva în loc de o mână obișnuită sau o lupă. În cele mai multe cazuri, cel mai bine este să lăsați lucrurile așa cum sunt.

    Treceți cu mouse-ul peste text (valorile proprietății) și cursorul își va schimba aspectul, exact ca în tabelul de mai jos.

Lucrul cu culoarea

  • Culoarea textului

    Proprietatea color poate fi folosită pentru a schimba culoarea textului unui element. Browserele moderne acceptă câteva moduri de bază de a descrie culorile: cuvinte cheie, RGB și HEX.

    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi, veritatis nihil alias iste odit similique sit eius optio veniam, impedit cumque fuga facere labore quo id necessitatibus quaerat rerum.

    HTML

    <p class="text">
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi, veritatis nihil alias iste odit similique sit eius optio veniam, impedit cumque fuga facere labore quo id necessitatibus quaerat rerum.
    </p>

    CSS

    .text {
    color: tomato;
    }

    Util !!!

    Pe site-ul htmlcolorcodes.com puteți vedea valorile culorilor în diferite formate.

  • Culoarea de fundal

    Proprietatea background-color permite să modificați culoarea de fundal a unui element.

    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi, veritatis nihil alias iste odit similique sit eius optio veniam, impedit cumque fuga facere labore quo id necessitatibus quaerat rerum.

    CSS

    .text-new {
    background-color: tomato;
    color: white;
    }

  • Nume de culori

    Pentru a descrie culorile în stiluri, puteți utiliza următoarele denumiri de culori. Acestea nu mai sunt folosite în dezvoltarea web modernă, dar sunt utile dacă trebuie să setați rapid o culoare.

  • RGB

    Există trei culori principale: roșu, verde și albastru. Amestecându-le în diferite proporții, puteți obține orice altă culoare.

    În acest format se folosește funcția rgb() pentru a seta valoarea care specifică proporția fiecăreia dintre cele trei culori primare, după care acestea sunt amestecate și se obține culoarea dorită.

    Intervalul de valori poate fi specificat sub formă de numere de la 0 (absența completă a culorii) la 255 (proporția maximă). Aceasta este o modalitate de bază de a seta o culoare RGB.

    /* Culoarea roșie pură: 255 roșu, 0 verde, 0 albastru */
    p {
    color: rgb(255, 0, 0);
    }

    Sau procentual de la 0% până la 100%.

    /* Culoarea roșie pură: 100% roșu, 0% verde, 0% albastru */
    p {
    color: rgb(100%, 0%, 0%);
    }

    Atentie !!!

    Nu este nevoie să memorați combinațiile valorilor RGB, toate culorile sunt în layoutul creat de designer.

  • HEX

    Această descriere a culorii se bazează pe notarea numărului în hexazecimal. Ponderea fiecăreia dintre cele trei culori primare este descrisă de două numere.

    RRGGBB

    Când vine vorba de utilizarea cifrelor în acest sistem de numere, se utilizează un set de la 0 la 9 și litere latine de la A la F cu litere mici sau mari.

    • 0 - cea mai mică fracțiune de culoare
    • f - cea mai mare fracțiune de culoare

    /* Culoarea roșie pură: ff (100%) roșu, 00 (0%) verde, 00 (0%) albastru */
    p {
    color: #ff0000;
    }

    /* Culoarea verde pură: 00 (0%) roșu, ff (100%) verde, 00 (0%) albastru */
    p {
    color: #00ff00;
    }

    /* Culoarea albastră pură: 00 (0%) roșu, 00 (0%) verde, ff (100%) albastru */
    p {
    color: #0000ff;
    }

    Atentie !!!

    Nu este nevoie să memorați combinațiile valorilor RGB, toate culorile sunt în layoutul creat de designer.

  • Transparența culorii

    Principala modalitate de a adăuga transparență unei culori este cu funcția rgba(). Spre deosebire de rgb(), este adăugată și transparența (canalul alfa).

    rgba(roșu, verde, albastru, alfa)

    În majoritatea cazurilor, transparența este dată de un număr de la 0 (transparent) la 1 (nu este transparent).

    /* Culoarea roșie pură cu transparență de 30% */
    p {
    background-color: rgba(255, 0, 0, 0.3);
    }

    Dar puteți folosi și procente de la 0% (transaprent) până la 100% (nu este transparent).

    /* Culoarea roșie pură cu transparență de 30% */
    p {
    background-color: rgba(100%, 0%, 0%, 30%);
    }

    Interesant !!!

    Transparența culorii poate fi setată și în formatul HEX, dar din cauza complexității unei astfel de notații și reținerii valorilor, mereu se folosește funcția rgba().

Cascada stilurilor

Cascada – este un mecanism care gestionează valorile finale ale proprietăților unui element atunci când îi sunt aplicate mai multe reguli CSS.

  • Dacă unui element i se aplică mai multe reguli, proprietățile acestora vor fi combinate.
  • Dacă regulile au aceleași proprietăți cu valori diferite, atunci acestea vor fi în dezacord.

    • /* Ce culoare va fi textul paragrafelor? */
    p {
    color: blue;
    background-color: yellow;
    }

    p {
    color: black;
    }

    Observatie !!!

    Ia culoarea ultimei comenzi date, daca se contrazic

    Pentru a colecta stilurile finale ale unui element și pentru a rezolva dezacordurile în valorile proprietăților, browserul folosește două mecanisme: specificitatea și moștenirea.

  • Specificitatea selectorului

    Pentru fiecare regulă CSS, browserul calculează specificitatea (greutatea) selectorului. Dacă un element necesită să i se aplice o proprietate din diferite reguli, dar există proprietăți în dezacord (aceleași), se folosește valoarea proprietății din regula cu cea mai mare specificitate a selectorului.

    Valoarea specificității e formată din patru ranguri, a căror importanță crește de la dreapta la stânga.

    inline<---ID<---classes,attributes,pseudo-classes<---elements,pseudo-elements

    • Selectorii de rangul 4

      Fiecare selector de element și pseudo-element adaugă o unitate la rangul 4.

      Culoarea textului paragrafului va fi portocaliu, deoarece specificitatea celei de-a doua reguli este mai mare.

      /* Specificitatea - 0 0 0 1 */
      p {
      color: green;
      }

      /* ✅ Specificitatea - 0 0 0 2 */
      article > p {
      color: orange;
      }

    • Selectorii de rangul 3

      Fiecare selector de clasă, atribut și pseudo-clasă adaugă o unitate la rangul 3.

      Exemplu

      Lorem ipsum dolor sit amet

      Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi, veritatis nihil alias iste odit similique sit eius optio veniam, impedit cumque fuga facere labore quo id necessitatibus quaerat rerum.

      Citeşte mai mult...
      HTML

      <article class="post">
      <h1 class="post-title">Lorem ipsum dolor sit amet</h1>
      <p>
      Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
      veritatis nihil alias iste odit similique sit eius optio veniam, impedit
      cumque fuga facere labore quo id necessitatibus quaerat rerum.
      </p>
      <a href="" class="post-link">Citeşte mai mult...</a>
      </article>

      CSS

      • Culoarea textului titlului va fi portocaliu, deoarece specificitate celei de-a treia reguli este mai mare.

        /* Specificitatea - 0 0 1 0 */
        .post-title {
        color: green;
        }

        /* Specificitatea - 0 0 1 1 */
        .post > h1 {
        color: red;
        }

        /* ✅ Specificitatea - 0 0 2 0 */
        .post > .post-title {
        color: orange;
        }

      • Culoarea textului titlului va fi maro, deoarece specificitatea celei de-a cincea reguli este mai mare.

        /* Specificitatea - 0 0 0 1 */
        a {
        color: green;
        }

        /* Specificitatea - 0 0 1 0 */
        .post-link {
        color: orange;
        }

        /* Specificitatea - 0 0 1 1 */
        a.post-link {
        color: blue;
        }

        /* Specificitatea - 0 0 2 0 */
        .post > .post-link {
        color: red;
        }

        /* ✅ Specificitatea - 0 0 2 1 */
        .post > a.post-link {
        color: brown;
        }

    • Selectorii de rangul 2

      Fiecare selector de ID adaugă o unitate la rangul 2.

      <h1 class="post-title" id="title">

      Culoarea textului titlului va fi portocaliu, deoarece specificitate celei de-a doua reguli este mai mare.

      /* Specificitatea - 0 0 1 0 */
      .post-title {
      color: green;
      }

      /* ✅ Specificitatea - 0 1 0 0 */
      #title {
      color: orange;
      }

    • Selectorii de rangul 1

      Stilurile de tip inline adaugă o unitate la rangul 1, au specificitatea cea mai mare și nu pot fi redefinite prin CSS.

      Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi, veritatis nihil alias iste odit similique sit eius optio veniam, impedit cumque fuga facere labore quo id necessitatibus quaerat rerum.

      HTML si CSS

      <p style="color: green">
      Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
      veritatis nihil alias iste odit similique sit eius optio veniam, impedit
      cumque fuga facere labore quo id necessitatibus quaerat rerum.
      </p>

    • Aceeași specificitate

      Dacă specificitatea este aceeași, plasată mai jos din cod e aplicată cu întâietate.

      /* Trei selectori de elemente cu aceeași specificitate */
      a {
      color: teal;
      }

      a {
      color: brown;
      }

      /* Întrucât această regulă se află mai jos, culoarea portocalie a textului va fi aplicată tuturor link-urilor*/
      a {
      color: orange;
      }

    • Evitarea specificității

      Dacă selectorul are forma A > B > C + X Y, ar trebui să luăm în considerare următoarele: Cu cât selectorul este mai simplu, cu atât vor apărea mai puține probleme de specificitate. Obișnuiți-vă să folosiți selectori de clasă. Atributul class nu numai că ajută la evitarea problemelor de specificitate, dar și îmbunătățește lizibilitatea marcajului HTML.

      Exemplu

      • Prima abordare este de a atribui o clasă unui bloc părinte comun și de a folosi selectorii de copii și descendenți.

        Lorem ipsum dolor sit amet

        Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi, veritatis nihil alias iste odit similique sit eius optio veniam, impedit cumque fuga facere labore quo id necessitatibus quaerat rerum.

        Citeşte mai mult...

        Acest CSS funcționează corect atunci când marcajul este simplu. Dar, pe măsură ce complexitatea marcajului blocurilor crește, stilizarea adecvată de către selectorii de tag-uri devine imposibilă. De exemplu, dacă în postare există 5 link-uri în loc de unul, selectarea punctuală a unui link nu va fi posibilă sau selectorul va fi prea complicat.

        CSS

        .post { /* Stiluri articol */
        }

        .pos
        t > h1 { /* Stiluri titlu */
        }

        .post > p {
        /* Stiluri text */
        }

        .post > a {
        /* Stiluri link */
        }

        .post > a:hover {
        /* Stilurile link-ului la hover */
        }

      • A doua abordare este să adăugați clase descriptive blocului părinte și tag-urilor din interioul acestuia.

        • article class="post"
        • h1 class="title"
        • p class="text"
        • a class="link"

        Acest CSS este scalat corect și bine suportat, fiind punctul de plecare pentru a scrie un CSS bun. Cu toate acestea, pe măsură ce complexitatea marcajului crește, pot apărea mici probleme de specificitate.

        CSS

        .post {
        /* Stiluri articol */
        }

        .post > .title {
        /* Stiluri titlu */
        }

        .post > .text {
        /* Stiluri text */
        }

        .post > .link {
        /* Stiluri link */
        }

        .post > .link:hover {
        /* Stilurile link-ului la hover */
        }

      • O a treia abordare este de a atribui clase foarte specifice blocului părinte și fiecărui tag din cadrul acestuia după principiul nume_părinte-nume_copil.

        • article class="post"
        • h1 class="post-title"
        • p class="post-text"
        • a href="" class="post-link"

        Această abordare folosește selectori de clasă simpli, fără a fi nevoie să ne gândim la specificitate. Un astfel de CSS se scalează bine și este suportat. Pentru scrierea stilurilor, dezvoltarea modernă folosește variații ale acestei abordări.

        CSS

        .post {
        /* Stiluri articol */
        }

        .post-title {
        /* Stiluri titlu */
        }

        .post-text {
        /* Stiluri text */
        }

        .post-link {
        /* Stiluri link */
        }

        .post-link:hover {
        /* Stilurile link-ului la hover */
        }

      Util !!!

      Nu vă limitați doar la o singură abordare. Toate cele trei metode de denumire a marcajului cu clase și stiluri de scriere pot fi folosite împreună, în funcție de situație.

    • Cuvântul cheie (!important)

      Specificitatea unei reguli poate fi mărită prin adăugarea cuvântului cheie !important după valoarea proprietății.

      Valoarea unei astfel de proprietăți are prioritate față de toate celelalte, iar culoarea textului paragrafului va fi portocaliu.

      p {
      color: orange !important;
      }

      p#text-id.text {
      color: blue;
      }

      Atentie...!!!

      Nu se recomandă deloc să folosiți !important în dezvoltarea modernă. Singurul caz rezonabil este să redefiniți valoarea unei proprietăți, dacă nu există acces direct la un fișier cu stiluri, cum ar fi un stil dintr-o bibliotecă.

  • Moștenirea proprietății

    Moștenirea genetică poate fi adesea observată în viața reală. De exemplu, părinții înalți au cel mai des copii înalți, cel mai des este moștenită culoarea părului etc. În CSS există ceva similar. Dacă setați culoarea textului părintelui în verde, atunci culoarea textului tuturor descendenților săi va fi, de asemenea, verde, cu excepția cazului în care o regulă va înlocui această valoare.

    Moștenirea reprezintă un mecanism prin care valorile unor proprietăți sunt transmise de la un strămoș la descendenții săi. În special sunt moștenite proprietățile de formatare a textului, cum ar fi culoarea. Acest lucru este convenabil deoarece nu trebuie să setați stiluri repetitive pentru fiecare copil în mod individual.

    HTML

    article class="post"

    CSS

    Haideți să setăm întregul text al postării în culoarea verde.

    .post {
    color: green;
    }

    În viață, nu toate atributele părinților sunt transmise copiilor. Același lucru este și în cazul cu CSS, nu toate proprietățile implicite sunt moștenite de descendenți. De fapt, dacă toate proprietățile ar fi moștenite, efectul ar fi similar cu lipsa moștenirii și ar trebui scris mult CSS pentru a reseta stilurile moștenite de la părinte ca să redefiniți acest comportament.

    • Excepții

      Nu toate elementele moștenesc valoarea proprietăților de la strămoșii lor. De exemplu, link-ul nu moștenește culoarea textului.

      Dacă doriți ca link-ul din postare să aibă aceeași culoare cu restul textului, puteți utiliza valoarea specială de moștenire inherit pentru proprietatea color, ca să evitați duplicarea culorii în două locuri.

      HTML

      article class="post"

      a href="" class="post-link"

      CSS

      .post {
      color: green;
      }

      .post-link {
      color: inherit;
      }

      Util...!!!

      Valoarea inherit îi comunică browserului să moștenească valoarea proprietății de la un strămoș. Pentru culoare este mai bine să folosiți valoarea currentColor (culoarea curentă).

    • Culoarea textului paginii

      Pentru a seta culoarea textului de bază (dominant) a întregului document, creăm o regulă pentru tag-ul (body) și adăugăm proprietatea color. Această culoare va fi moștenită de tot conținutul text, cu excepția link-urilor.

      body {
      color: #2a2a2a;
      }

Variabile CSS

Proprietățile personalizate (CSS Custom Properties), cunoscute și sub numele de variabile CSS, reprezintă containere pentru valorile definite într-un document CSS pentru o utilizare repetată. Variabilele CSS facilitează utilizarea și modificarea valorilor repetate într-o foaie de stil. Acestea sunt acceptate nativ de browser și nu necesită instrumente suplimentare.

Exemplu

Section title

HTML

<section class="section">
<h1 class="section-title">Section title</h1>
</section>

CSS

.section { /* Alte stiluri */
border: 2px solid #3f51b5;
}

.section-title {
/* Alte stiluri*/
color: #3f51b5;
}

.section-title::before {
/* Alte stiluri */
background-color: #3f51b5;
}

În exemplul de mai sus, valoarea #3f51b5 apare de trei ori. Să presupunem că aceasta este culoarea de bază a mărcii, care este utilizată în mod constant pe diferite pagini ale site-ului. Dacă culoarea mărcii se schimbă, va trebui să înlocuim această valoare peste tot, astfel într-un proiect mare, aceasta poate fi în sute de locuri, iar cea mai bună opțiune este să folosim căutarea și înlocuirea.

  • Declararea unei variabile

    Variabilele CSS ne vin în ajutor. Numele variabilei ține cont de majuscule și minuscule și poate conține litere, cifre, litere de subliniere și liniuțe. Haideți să creăm o variabilă --brand-color pentru a păstra culoarea mărcii (#3f51b5) și să o folosim în cod.

    CSS

    :root {
    --brand-color: #3f51b5;
    }

    Variabilele globale disponibile în orice altă regulă sunt definite în selectorul :root, un selector mai specific al elementului (html). Declararea variabilei începe cu o liniuță dublă -- urmată de numele acesteia în notație arbitrară. De exemplu, brand-color sau brandColor sunt nume de variabile valide, atâta timp cât toate variabilele din proiect au aceeași notație. Numele și valoarea sunt separate prin două puncte, iar la sfârșitul declarării se pune punct și virgulă.

    Interesant...!!!

    Declararea unei variabile înseamnă crearea unei noi proprietăți CSS care este moștenită și, prin urmare, este disponibilă peste tot, deoarece orice element este un descendent al elementului (html).

  • Utilizarea unei variabile

    Pentru a accesa valoarea unei variabile, se folosește funcția încorporată var(), care primește un nume de variabilă ca prim argument.

    CSS

    :root {
    --brand-color: #3f51b5;
    }

    .section {
    /* Alte stiluri */
    border: 2px solid var(--brand-color);
    }

    .section-title {
    /* Alte stiluri */
    color: var(--brand-color);
    }

    .section-title::before {
    /* Alte stiluri */
    background-color: var(--brand-color);
    }

    În cazul dat, nu a apărut mai mult cod, dar este mai flexibil și mai ușor de citit. Dacă va trebui să schimbați culoarea mărcii, va fi de ajuns să modificați doar valoarea variabilei --brand-color, prin urmare, culoarea se va schimba oriunde este folosită.

    Util...!!!

    În cazul dat, nu a apărut mai mult cod, dar este mai flexibil și mai ușor de citit. Dacă va trebui să schimbați culoarea mărcii, va fi de ajuns să modificați doar valoarea variabilei --brand-color, prin urmare, culoarea se va schimba oriunde este folosită.

  • Valorile de rezervă

    Funcția var(variable_name, fallback) poate lua două valori. Al doilea argument este alternativ – valoarea care va fi folosită dacă variabila transmisă în primul argument este nedefinită.

    CSS

    .section {
    border: 2px solid var(--brand-color, orange);
    }

    .section-title {
    color: var(--brand-color, black);
    }

    .section-title::before { background-color: var(--brand-color, tomato);
    }

    Util...!!!

    În cazul când se utilizează variabile CSS, capacitatea de a transmite o valoare implicită este în primul rând o opțiune, nu o cerință. Nu este nevoie să specificați peste tot o rezervă.

  • Domeniul de aplicare

    Variabila --brand-color este globală deoarece este definită în :root, adică este disponibilă pe orice alt element. Dacă valoarea va fi folosită doar într-o anumită parte a paginii, puteți crea variabile cu un domeniu local. Acestea vor fi disponibile doar pentru elementul și descendenții săi în care sunt declarate.

    Exemplu

    Alert text

    HTML

    <div class="alert">>
    <p class="alert-text">Alert text</p>
    <svg class="alert-icon"></svg>
    </div>

    CSS

    .alert {
    --alert-text-color: black;
    --alert-accent-color: tomato;
    }

    .alert-text {
    color: var(--alert-text-color);
    border: 1px solid var(--alert-accent-color);
    }

    .alert-icon {
    background-color: var(--alert-accent-color);
    }

  • Redefinirea valorilor

    Variabilele pot fi redefinite local cu alte valori. Pentru a face acest lucru, trebuie să declarați o variabilă cu același nume ca în strămoș și să-i setați valoarea necesară.

    • Default
    • Success
    • Warning
    • Error

    HTML

    li class="alert">Default/li

    li class="alert success">Success/li

    li class="alert warning">Warning/li

    li class="alert error">Error/li

    CSS

    body {
    font-family: sans-serif;
    line-height: 1.5;
    }

    .alert {
    --alert-text-color: black;

    color: var(--alert-text-color);
    }

    .alert.success {
    --alert-text-color: green;
    }

    .alert.warning {
    --alert-text-color: orange;
    }

    .alert.error {
    --alert-text-color: red;
    }
